# rxrpl ↔ rippled interop

Mixed-network gating for rxrpl. Spins up a private XRPL network of three
`rippled` validators and two `rxrpl` validators, all sharing a common UNL
and genesis ledger, and runs a Python pytest suite that:

| Suite | What it checks |
|---|---|
| `propagation` | A payment submitted on rippled appears on rxrpl (and vice versa) within a few ledgers. Exercises P2P transaction relay. |
| `consensus` | All five nodes advance past a target ledger and agree on the same hash for a fully-validated ledger. Exercises the full RPCA + UNL trust stack. |
| `sync` | A late-joining rxrpl node catches up to a network with prior activity. Exercises GetLedger / TMLedgerData (rippled wire format). |

## Wire-format dependency

The interop suite cannot pass without **PR #15** (rippled wire format
compatibility). That PR drops the rxrpl-specific 32-byte content-hash
shortcut from the GetLedger delta protocol and unifies the codebase on
rippled's 33-byte `(path, depth)` `NodeId` representation. Until that
PR is in `main`, the `sync` suite will hang at ledger 2 because the
rippled side rejects the malformed `node_ids` field and returns nothing
matchable.

## How to run locally

```bash
# Generate per-validator configs (deterministic, ed25519 seeds)
python3 interop/scripts/generate_configs.py

# Build rxrpl image once
docker build -t rxrpl:interop -f Dockerfile .

# Bring the network up + run the suite
./interop/scripts/run_interop.sh --suite all

# Or just the consensus suite
./interop/scripts/run_interop.sh --suite consensus

# Cleanup
docker compose -f interop/docker-compose.yml down -v
```

## CI cadence

GitHub Actions (`.github/workflows/interop.yml`):

- **Nightly** (02:00 UTC, Mon–Sun) – the full suite, `RIPPLED_IMAGE=rippleci/rippled:2.3.0`.
- **Manual dispatch** – choose suite + image via the Actions UI.

The job is informational (it does not block PRs), but a green nightly
run is the gate for declaring rxrpl rippled-compatible at the wire layer.

## Files

- `docker-compose.yml` – the 5-node mixed network.
- `configs/` – per-node TOML/cfg files. Regenerated by `generate_configs.py`.
- `tests/` – pytest suite, run inside the `test-runner` container via `Dockerfile.test`.
- `scripts/run_interop.sh` – local entry point.
- `scripts/quick_handshake.sh` – minimal 2-node sanity check (1 rippled + 1 rxrpl) for fast iteration.
